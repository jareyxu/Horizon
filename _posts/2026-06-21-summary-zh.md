中国工信部已完成《智能网联汽车自动驾驶系统安全要求》强制性国家标准报批稿，建议 2027 年 7 月 1 日实施。该标准引入了安全档案机制(Safety Case)，针对 L3 和 L4 级自动驾驶提出了严格的安全要求。 这是我国首部针对 L3/L4 级自动驾驶的强制性国家标准，标志着监管从‘概念松绑’转向‘安全硬约束’。它将终结车企靠模糊宣传抢市场的阶段，未来竞争聚焦安全能力，短期内可能推高成本但长期促进技术迭代。 该标准引入了安全档案机制(Safety Case)，要求企业用‘声明—论据—证据’系统性论证安全性。此外，对 L3 级的人机交接和 L4 级的系统自主风险处置分别提出了要求。

## [三维光纤微镊力达传统光镊十万倍](https://www.stdaily.com/web/gdxw/2026-06/19/content_534836.html) ⭐️ 9.0/10

Cloudflare 推出了临时 Worker 帐户，允许 AI 代理和开发者通过 'wrangler deploy --temporary' 部署 Worker，创建持续 60 分钟的实时部署。这些临时帐户可以被认领成为永久帐户，否则自动过期。 此功能移除了 AI 代理的人类身份验证障碍，实现了完全自动化的临时部署，可用于测试、PR 预览和 AI 驱动的工作流。它还为用户提供了免费的 60 分钟临时部署，对实验非常有价值。 临时部署持续 60 分钟，之后过期，除非通过永久 Cloudflare 帐户认领。Cloudflare 应用了速率限制和滥用预防检查以防止滥用，例如限制创建临时预览帐户的速度。

## [IETF 提议新增 HTTP QUERY 方法以支持带请求体的安全查询](https://httpwg.org/http-extensions/draft-ietf-httpbis-safe-method-w-body.html) ⭐️ 8.0/10

IETF HTTP 工作组正在制定一项草案，拟新增 HTTP QUERY 方法，允许将查询条件放在请求体中，同时保持与 GET 方法相同的安全性和幂等性，便于缓存、重试和自动恢复。草案还定义了新的 Accept-Query 响应头，用于服务器声明支持的查询格式。 该提案解决了 URI 长度限制这一长期问题，允许将复杂查询通过请求体发送，这对于具有大型查询负载的现代 Web 应用至关重要。与基于 POST 的变通方法相比，它还提高了可缓存性和可靠性，可能影响 API 和 Web 服务处理查询操作的方式。 QUERY 方法具有安全性和幂等性，意味着它不会改变服务器状态，重复请求产生相同结果，从而支持自动重试和缓存。当前草案（版本 -14）将于 2026 年 12 月 22 日到期，工作组内部仍在积极讨论中。

telegram · zaihuapd · 6月20日 06:28

**背景**: HTTP GET 请求通过 URI 查询字符串传递参数，但其长度受限于客户端和服务器的限制（通常为 2-8 KB）。POST 请求可以携带请求体，但默认不具有安全性和幂等性，因此不适合缓存和自动重试。拟议的 QUERY 方法填补了这一空白，结合了 GET 的安全性与幂等性以及发送请求体的能力，类似于某些 API 已经使用 POST 进行查询的方式，但缺乏标准化的保证。
